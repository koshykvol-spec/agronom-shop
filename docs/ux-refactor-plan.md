# UI/UX рефакторинг «Агроном» — план

Складено за зовнішнім аудитом власника + верифікацією 8 продуктових лінз (агенти читали реальний код).
Філософія: інкрементально, ванільний JS, free tier; нове — кероване через адмінку (D1), не хардкод.
**Не ламати** миттєвий клієнтський пошук (fuzzy/синоніми працюють бо весь каталог у пам'яті).

## Критика аудиту (ключове)
- Підтверджено ~80% тверджень.
- Перебільшено: «/api/products 850KB» — це fallback `products.json`, живий API ~50-150KB; «фільтри в bottom sheet» — фільтрів нема; «нема summary» — є, але йде в Telegram (юзер не бачить).
- ⛔ Проти SSR-пагінації каталогу (audit #7): зламає клієнтський пошук, виграш при 2k мінімальний. Замінено на payload-оптимізацію.
- Пропущено: варіант-матриця фасувань, консультація-CTA, адреса магазину на PDP, калькулятор дози, рейтинг на картках, seasonal→кошик-набір, :focus-visible/dialog-roles.
